## Запуск

```bash
./run.sh build_generator
./run.sh run_generator
./run.sh build_reporter
./run.sh run_reporter
```

После этого в `data/` появятся `data.csv` и `report.html`.

## Как открыть отчёт в браузере (GitHub Codespaces)
Цепочка проброса до браузера на компьютере состоит из трёх уровней:

1. nginx слушает порт 80 внутри контейнера;
2. `docker run -p 8080:80` публикует этот порт на хосте Codespaces (облачная
   виртуальная машина) как порт 8080;
3. Codespaces обнаруживает слушающий порт 8080 и пробрасывает его на компьютер.
   VS Code поднимает локальный туннель и адрес доступен как `127.0.0.1:8080`
   на компьютере; при публичной видимости порт доступен ещё и по адресу вида
   `https://<codespace>-8080.app.github.dev`. Оба варианта ведут через одну и ту
   же цепочку.

Порядок действий:

1. Открыть репозиторий в GitHub Codespaces.
2. Выполнить запуск (см. выше), чтобы получить `data/report.html`.
3. Запустить веб-сервер:

   ```bash
   ./run.sh report_server
   ```
4. В правом нижнем углу появится всплывающее окно, в нем нажать **Open in Browser**. Если не появилось, то идем дальше:
5. В VS Code открыть вкладку **PORTS**. Порт 8080 пробрасывается автоматически.
   Если его нет — нажать **Forward a Port** и указать `8080`.
6. В строке порта 8080 скопировать **Forwarded Address** (адрес вида
   `127.0.0.1:8080`) или нажать на значок глобуса (**Open in Browser**).
   Отчёт откроется в браузере.
